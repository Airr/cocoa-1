Performance Tests

Alder Kmer provides you with a simple user-interface to help you count Kmers in a sequence file. The steps includes:

  • Load a sequence file.

  • Adjust disk space, memory size, and CPU cores.

  • Choose the size K or the length of fixed-length DNA sequences.

  • Select a folder or a directory with a prefix for the output files.

  • Start counting Kmers.

  • Wait for the counting to finish. It could take a while.

  • Search the count table for the frequency of a Kmer.

Let us following the detail of the procedure of running Alder Kmer.

Load a sequence file in FASTA or FASTQ format using Run>Load menu (COMMAND+L) or by clicking a button next the text input box of Input file. Pushing the button with an up-arrow, you would be invited to navigate the file system in your computer to choose a sequence file. As sequence files in counting Kmers tend to be huge, and they are often compressed, Alder Kmer can take gzipped FASTA or FASTQ sequence files. The size of the input file is shown just below the file name that you have selected.

Alder Kmer displays available disk space (in giga-byte), free memory (in mega-byte), and the total number of CPU cores. Use slide bars to adjust the size of disk space and memory that would be allocated to Alder Kmer. You can type in numbers instead. Note that the number in disk space is in giga-byte, and that of memory is in mega-byte. You can also change the number of CPU cores that Alder Kmer would use.

Choose a length K of Kmer sequences. The length K may depends on the allocated disk space and the memory for Alder Kmer. Select an output directory and a prefix for the output files. You can use a button with an up-arrow to select a directory, or type in the directory name. Type in an output file name prefix.

Now, you are ready to count Kmer sequences. Push the button named "Count" or Run>Count menu (COMMAND+K). The progress of a run is indicated in the lower left corner.

After a run is done, the summary of the execution would be displayed. These include the maximum number of Kmers in the input file, the number of iterations over partition files, the partition files size, the counting table file name and size, and the number of unique Kmers in the counting table.

You may search the counting table for the number of a Kmer in the input sequence file. Type in a Kmer sequence, and push a button named "Search" or select Run>Search menu (COMMAND+F).
